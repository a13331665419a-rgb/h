# 解说漫剧本画面任务拆解规则｜自检机制版

> 适用方向：AI 解说漫、AI 小说推文视频、AI 短剧旁白驱动画面拆解、可灵生图 / 图生视频提示词、Seedance / Vidu / 即梦类视频模型提示词前置拆解。  
> 核心目标：拿到解说漫剧本后，不按“每句话一镜”机械拆分，而是按“旁白节奏 + 情绪强度 + 画面任务 + AI 可执行性”拆解成可稳定生成的视频提示词。

# 1. 核心原则

解说漫剧本不能按“每句话一镜”机械拆。

正确逻辑是：

```text
先读懂旁白内容
↓
判断这段旁白承担什么画面任务
↓
决定需要几个画面表现
↓
把抽象情绪转成可见动作
↓
保证人物站位、场景、道具、情绪连续
↓
再生成生图提示词和图生视频提示词
```

## 1.1 一镜必须有明确任务

每个画面都必须回答一个问题：

```text
这一镜是在建立场景？
还是在建立人物状态？
还是在放大情绪？
还是在制造关系压迫？
还是在揭露信息？
还是在给观众反应时间？
还是在服务转场？
还是在制造爆点卡点？
```

如果一镜无法回答这个问题，就说明它是无效镜头。

## 1.2 不要把旁白直接翻译成画面

错误逻辑：

```text
旁白：我很痛苦。
画面：女主很痛苦。
```

正确逻辑：

```text
旁白：我很痛苦。
画面：女主低头忍泪，手指死死攥住床单，嘴唇轻微发抖，肩膀压低，眼神避开男主。
```

## 1.3 画面要服务旁白节奏

解说漫画面不是独立短片，而是旁白的视觉承载。  
画面切换必须跟随：

```text
信息点
情绪点
转折点
悬念点
爽点
虐点
付费卡点
```

# 2. 标准工作流

拿到解说漫剧本后，按以下流程执行：

```text
第 1 步：通读全文，判断类型与核心情绪
第 2 步：按标点、转折词、情绪爆点拆分旁白
第 3 步：为每条旁白判断画面任务
第 4 步：决定每条旁白需要 1 个、2 个还是 3 个画面
第 5 步：生成画面信息表
第 6 步：进行第一轮自检：原文忠实度、信息覆盖度
第 7 步：补齐缺失画面、删除无效画面
第 8 步：生成生图提示词
第 9 步：进行第二轮自检：角色、场景、站位、构图、光影
第 10 步：生成图生视频提示词
第 11 步：进行第三轮自检：动作复杂度、镜头稳定性、连续性
第 12 步：输出最终可用版本
```

# 3. 旁白拆分规则

## 3.1 按标点拆

优先按以下标点拆分基础旁白单元：

```text
逗号
句号
分号
问号
感叹号
冒号
破折号
省略号
```

示例：

```text
原文：我躺在急诊室里，刺鼻的消毒水味几乎让我窒息。
```

拆分为：

```text
1. 我躺在急诊室里
2. 刺鼻的消毒水味几乎让我窒息
```

## 3.2 按转折词拆

遇到以下词语，必须优先拆开：

```text
但是
可是
然而
没想到
直到
偏偏
真正
却
竟然
原来
此刻
下一秒
就在这时
```

示例：

```text
原文：我以为他会救我，可他却亲手把我推进深渊。
```

拆分为：

```text
1. 我以为他会救我
2. 可他却亲手把我推进深渊
```

第一段是期待，第二段是背叛反转，不能放在同一镜里。

## 3.3 按信息点拆

如果一句旁白中包含多个剧情信息，必须拆开。

示例：

```text
原文：昨晚的车祸明明是 Samantha 酒后无证驾驶造成的，可 Rhett 却向老师作证，说开车的人是我。
```

拆分为：

```text
1. 昨晚的车祸
2. Samantha 酒后无证驾驶
3. 撞上学校大门
4. Rhett 向老师作证
5. 他指认我是司机
```

每个信息点都必须有画面承载。

## 3.4 按情绪变化拆

如果一句旁白从一种情绪转到另一种情绪，必须拆开。

常见情绪变化：

```text
期待 → 失望
信任 → 背叛
委屈 → 愤怒
害怕 → 反抗
震惊 → 崩溃
羞辱 → 反杀
暧昧 → 危险
```

# 4. 画面任务分类

所有画面必须归入以下任务类型之一。

| 画面任务 | 主要作用 | 常见使用场景 |
|---|---|---|
| 场景建立 | 让观众知道地点、时间、氛围 | 医院、学校、宴会、车祸现场、狼群大厅 |
| 人物状态建立 | 让观众知道角色当前身体和心理状态 | 受伤、虚弱、被羞辱、刚醒来、濒临崩溃 |
| 情绪放大 | 把抽象情绪变成可见动作 | 忍泪、发抖、攥紧衣角、眼神失焦 |
| 关系压迫 | 展示权力差、距离感、压迫感 | 男主俯视女主、众人围攻、家人冷眼 |
| 反应切镜 | 给观众看到角色接收信息后的变化 | 听到背叛、发现真相、看到证据 |
| 道具/证据细节 | 用物件承载剧情信息 | 手机、合同、病历、车钥匙、录音、血迹 |
| 动作推进 | 推动剧情向下一步发展 | 推门、走近、转身、递文件、抬头 |
| 爆点/反转/卡点 | 强化爽点、虐点、悬念、付费点 | 身份曝光、背叛揭露、威胁、打脸 |
| 转场承接 | 保证镜头之间流畅过渡 | 视线、声音、光影、道具、动作匹配切 |

# 5. 画面数量判断规则

不是每条旁白都只生成一个画面。  
根据画面任务判断画面数量。

| 旁白类型 | 建议画面数量 | 拆解方式 |
|---|---:|---|
| 普通环境信息 | 1 个 | 场景建立 |
| 普通动作信息 | 1 个 | 动作推进 |
| 普通人物状态 | 1 个 | 人物状态建立 |
| 情绪句 | 2 个 | 主画面 + 情绪放大 |
| 关系冲突句 | 2 个 | 关系压迫 + 反应切镜 |
| 信息揭露句 | 2–3 个 | 信息画面 + 证据细节 + 反应 |
| 爆点反转句 | 3 个 | 铺垫 + 揭露 + 定格 |
| 转场句 | 1 个 | 道具 / 动作 / 视线承接 |

## 5.1 普通信息句

只需要一个画面。

```text
旁白：她走进教室。
画面：女主推开教室门，教室里的学生纷纷回头看她。
```

## 5.2 情绪句

至少两个画面。

```text
旁白：我难过得快要喘不过气。
画面 1：女主站在人群边缘，身体微微后退。
画面 2：女主手指攥紧衣角，眼眶泛红，嘴唇发抖。
```

## 5.3 爆点句

必须三步处理。

```text
铺垫：女主听到某个信息后僵住。
揭露：男主 / 反派 / 证据完成关键信息揭露。
定格：女主或观众最需要记住的情绪特写。
```

# 6. 每类画面任务拆解标准

## 6.1 场景建立任务

必须包含：

```text
地点
时间
环境元素
主角位置
背景人物或背景运动
光影基调
整体氛围
```

错误：

```text
女主在医院。
```

正确：

```text
冷白急诊室里，女主半躺在病床上，头上缠着渗血绷带，输液架立在画面右侧，急诊帘半开，背景护士虚化经过，刺眼顶灯从上方压下来，整体氛围冷白、压抑、窒息。
```

## 6.2 人物状态建立任务

必须包含：

```text
身体状态
表情状态
手部动作
呼吸状态
姿态
服装 / 伤口 / 特殊标志
```

错误：

```text
女主很虚弱。
```

正确：

```text
女主脸色苍白，嘴唇干裂，肩膀细微发抖，右手死死攥住白色床单，头上的绷带边缘渗出血迹，眼神失焦却强撑着不哭。
```

## 6.3 情绪放大任务

禁止只写抽象情绪。

禁止：

```text
她很痛苦。
她很绝望。
她很伤心。
她很害怕。
她很委屈。
```

必须视觉化：

```text
她低头忍泪，睫毛轻颤，手指攥紧衣角，嘴唇轻轻发抖，眼神避开所有人的注视，肩膀向内收缩。
```

优先使用以下身体细节：

```text
眼神
睫毛
嘴角
嘴唇
手指
肩膀
呼吸
站姿
身体退缩
眼泪欲落未落
```

## 6.4 关系压迫任务

必须写清人物空间关系：

```text
谁高位
谁低位
谁靠近
谁后退
谁俯视
谁被围住
谁掌控空间
谁被空间压迫
```

示例：

```text
女主坐在病床左下方，身体低位虚弱；男主站在床尾右侧，身形挺直，冷漠俯视她，两人之间隔着病床尾栏，形成明显的情感距离和权力压迫。
```

## 6.5 反应切镜任务

反应切镜不是重复剧情，而是呈现角色“接收信息后的变化”。

适合表现：

```text
听到背叛
听到真相
听到威胁
听到羞辱
看到证据
发现秘密
```

常用画面：

```text
女主瞳孔微缩
男主避开视线
反派嘴角轻轻上扬
母亲眼神动摇
围观者低声议论
主角手指突然停住
```

## 6.6 道具 / 证据细节任务

道具必须服务剧情，不允许随便添加。

可使用道具：

```text
手机屏幕
聊天记录
血迹
车钥匙
合同
录音笔
病历单
戒指
项链
破碎照片
门牌
校徽
酒杯
```

写法示例：

```text
手机屏幕特写，聊天记录停在“是她开的车”这一行，屏幕冷光映在女主苍白的脸上，她的手指悬在屏幕边缘，微微颤抖。
```

## 6.7 动作推进任务

动作必须简单、清楚、可生成。

适合 AI 的动作：

```text
抬头
低头
转身
走近
后退
推门
坐起
伸手
攥紧
松开
递出文件
看向某人
避开视线
```

不适合 AI 的动作：

```text
复杂打斗
多人混乱推搡
大幅奔跑追逐
复杂拥抱转身
摔倒翻滚
多人同步复杂动作
```

## 6.8 爆点 / 反转 / 卡点任务

爆点镜头必须三步走：

```text
1. 铺垫：观众意识到不对劲
2. 揭露：核心信息被说出或被看见
3. 定格：情绪特写或证据特写，形成停顿
```

示例：

```text
旁白：可真正陷害我的人，竟然是我最爱的人。

画面 1：女主听到真相后僵住，病房声音仿佛消失。
画面 2：男主冷漠站在床尾，避开她的眼神，没有任何愧疚。
画面 3：女主眼泪终于落下，画面定格在她不可置信的眼神上。
```

## 6.9 转场承接任务

每个大段落之间必须设计转场，不要硬切。

常用转场方式：

| 转场方式 | 用法 |
|---|---|
| 视线转场 | 女主看向门口 → 下一镜门被推开 |
| 声音转场 | 心电仪滴答声 → 车祸警报声 |
| 道具转场 | 血迹特写 → 车祸现场红色警灯 |
| 动作转场 | 手指攥紧床单 → 手握方向盘 |
| 光影转场 | 医院冷白灯 → 回忆中车灯强光 |
| 情绪转场 | 女主忍泪 → 回忆中崩溃大哭 |

# 7. 画面连续性与站位规则

解说漫虽然不是传统连续表演短剧，但人物站位必须稳定，否则 AI 视频会乱。

## 7.1 每个场景先建立空间轴线

进入新场景时，必须先定义：

```text
主角在画面哪一侧
对手在画面哪一侧
道具在哪一侧
门 / 窗 / 床 / 桌子的位置
镜头主要从哪一边拍
```

示例：

```text
医院急诊室空间轴线：
Scarlett 始终位于画面左下方病床上。
Rhett 始终站在床尾右侧。
输液架位于 Scarlett 右侧。
急诊门位于画面后方偏左。
镜头主要从 Scarlett 斜前方或床尾方向拍摄，不随意越轴。
```

## 7.2 禁止无理由换位

错误：

```text
镜头 1：男主站在女主右侧。
镜头 2：男主突然出现在女主左侧。
镜头 3：女主突然站到男主背后。
```

正确：

```text
除非剧情明确发生走位，否则同一场景内人物位置保持稳定。
```

## 7.3 反打镜头也要保持方向一致

如果镜头从女主特写切到男主特写，要保持视线方向。

```text
女主看向画面右上方。
男主看向画面左下方。
```

这样观众能理解两人正在对视。

# 8. AI 生成友好规则

## 8.1 优先微动作，不要复杂动作

AI 解说漫视频更适合：

```text
轻微抬眼
手指收紧
眼眶泛红
呼吸变急
嘴唇发抖
肩膀下沉
缓慢转头
轻微后退
```

不适合：

```text
快速打斗
复杂多人交互
高难度奔跑
多人同步动作
大幅度身体旋转
极端镜头翻转
```

## 8.2 每条视频提示词只允许一个核心动作

错误：

```text
女主坐起、哭泣、推开男主、冲出病房、回头大喊。
```

正确：

```text
女主挣扎着想坐起，右手撑住床边，眼神从震惊转为愤怒。
```

## 8.3 镜头运动要克制

推荐：

```text
缓慢推进
轻微拉近
固定镜头
轻微横移
轻微下压
慢慢虚化背景
```

不推荐：

```text
快速环绕
大幅摇镜
复杂航拍
突然变焦
旋转镜头
```

## 8.4 情绪靠表演和构图，不靠大动作

解说漫的强情绪主要靠：

```text
特写
眼神
手部细节
光影反差
人物距离
高低位压迫
背景虚化
声音节奏
```

# 9. 生图提示词生成规则

生图提示词的任务是生成稳定关键帧。

每条生图提示词必须包含：

```text
1. 画幅比例
2. 风格
3. 角色外貌
4. 场景
5. 人物站位
6. 动作
7. 表情
8. 景别
9. 构图
10. 机位
11. 光影
12. 情绪氛围
13. 禁止项
```

标准结构：

```text
竖屏 9:16，AI 解说漫，2.5D-3D cinematic short drama style，北美短剧质感，电影级光影。

【角色】
写清角色年龄、外貌、发型、服装、气质。

【场景】
写清地点、时间、环境、道具。

【画面内容】
写清人物正在做什么，谁站在哪里，谁看向谁。

【动作与表情】
写清手部、眼神、嘴角、肩膀、身体姿态。

【镜头】
写清景别、构图、机位、焦距倾向。

【光影】
写清冷暖、明暗、对比、饱和度、氛围。

【禁止项】
不要文字，不要水印，不要多余人物，不要变脸，不要变装，不要畸形手指，不要卡通感。
```

# 10. 图生视频提示词生成规则

图生视频提示词的任务不是重新讲剧情，而是让关键帧轻微动起来。

每条图生视频提示词只写：

```text
一个核心动作
一个情绪变化
一个镜头运动
一个环境动态
一组禁止项
```

标准结构：

```text
保持角色外貌、服装、场景、人物站位完全一致。

角色进行轻微自然动作：
写一个核心动作。

情绪变化：
写眼神、呼吸、嘴角、肩膀、手指的细微变化。

镜头运动：
缓慢推进 / 轻微拉近 / 横向微移 / 轻微下压 / 固定镜头。

环境动态：
灯光轻微闪烁 / 窗帘轻动 / 输液管轻晃 / 背景人影虚化经过。

禁止：
不要改变角色脸，不要改变服装，不要新增人物，不要大幅度动作，不要镜头乱转，不要卡通感。
```

# 11. 自检机制总览

本规则加入四轮自检机制。

```text
第一轮：旁白拆解自检
检查是否漏拆、错拆、过度合并、过度拆碎。

第二轮：画面任务自检
检查每个画面是否有明确任务，是否服务原文。

第三轮：连续性自检
检查人物站位、场景、道具、转场是否连续。

第四轮：AI 可执行性自检
检查生图和图生视频提示词是否稳定、清楚、可生成。
```

自检不是最后才做，而是每完成一个阶段就检查一次。

# 12. 分阶段自检清单

## 12.1 第一轮：旁白拆解自检

检查问题：

```text
1. 是否按标点拆分？
2. 是否按转折词拆分？
3. 是否按情绪变化拆分？
4. 是否按信息点拆分？
5. 是否有重要信息被合并到一个过长画面里？
6. 是否把一句很短但情绪很重的旁白忽略了？
7. 是否出现旁白编号混乱？
8. 是否保留了原文顺序？
```

修正原则：

```text
信息多的句子要拆开。
情绪重的短句要放大。
转折句必须单独成镜。
不能改变原文顺序。
```

## 12.2 第二轮：画面任务自检

每个画面都必须回答：

```text
这一镜的画面任务是什么？
```

检查问题：

```text
1. 每个画面是否标注了画面任务？
2. 是否存在没有任务的空镜？
3. 情绪句是否有情绪放大画面？
4. 爆点句是否有铺垫、揭露、定格？
5. 道具镜头是否真的服务剧情？
6. 反应切镜是否展示了角色变化？
7. 转场镜是否能自然接到下一镜？
```

如果某个画面不能归入任何任务类型，应删除或重写。

## 12.3 第三轮：原文忠实度自检

检查问题：

```text
1. 是否新增了原文没有的人物？
2. 是否新增了原文没有的关键道具？
3. 是否改变了人物关系？
4. 是否提前透露了后文信息？
5. 是否把旁白没说的动机强加给角色？
6. 是否把事实性信息改错？
7. 是否遗漏了旁白中的关键信息点？
```

原则：

```text
可以补充可视化细节。
不可以新增剧情事实。
可以强化情绪动作。
不可以改变角色动机。
可以设计转场。
不可以提前剧透。
```

## 12.4 第四轮：情绪视觉化自检

检查问题：

```text
1. 是否还存在“她很痛苦”“他很冷漠”这类抽象词？
2. 情绪是否变成了眼神、手指、呼吸、站姿、嘴角等可见动作？
3. 情绪变化是否有递进？
4. 画面情绪是否和旁白情绪一致？
5. 情绪是否过度夸张，变成不适合 AI 的大动作？
```

抽象词替换表：

| 抽象情绪 | 替换为可见动作 |
|---|---|
| 痛苦 | 手指攥紧、肩膀发抖、眉心压紧、呼吸变浅 |
| 绝望 | 眼神失焦、身体后退、手慢慢垂下 |
| 委屈 | 眼眶泛红、嘴唇发抖、低头忍泪 |
| 愤怒 | 下颌绷紧、眼神抬起、手指收紧 |
| 害怕 | 身体后缩、视线闪躲、呼吸急促 |
| 冷漠 | 眼神不动、嘴角无表情、身体不靠近 |
| 震惊 | 瞳孔微缩、动作停住、嘴唇微张 |

## 12.5 第五轮：站位与越轴自检

检查问题：

```text
1. 同一场景内人物位置是否稳定？
2. 角色是否无理由左右互换？
3. 反打镜头的视线方向是否正确？
4. 主角和对手的高低位关系是否明确？
5. 道具位置是否连续？
6. 场景入口、门、床、桌子等空间点是否混乱？
```

修正方式：

```text
为每个场景建立空间轴线。
每个镜头都写清谁在画面左侧，谁在画面右侧。
反打镜头保留视线方向。
除非剧情需要，不改变人物位置。
```

## 12.6 第六轮：节奏与时长自检

检查问题：

```text
1. 每个画面是否有预计时长？
2. 旁白长短是否和画面数量匹配？
3. 情绪重的地方是否留出停顿？
4. 信息密集处是否切得太快？
5. 爆点前是否有铺垫？
6. 爆点后是否有定格？
```

参考时长：

| 画面类型 | 建议时长 |
|---|---:|
| 场景建立 | 2–3 秒 |
| 人物状态 | 2–3 秒 |
| 情绪特写 | 2–4 秒 |
| 关系压迫 | 3–4 秒 |
| 证据细节 | 2–3 秒 |
| 爆点定格 | 3–5 秒 |
| 转场镜 | 1–2 秒 |

## 12.7 第七轮：生图提示词自检

检查问题：

```text
1. 是否写明竖屏 9:16？
2. 是否写明风格？
3. 是否写明角色外貌和服装？
4. 是否写明场景和道具？
5. 是否写明人物站位？
6. 是否写明动作和表情？
7. 是否写明景别、构图、机位？
8. 是否写明光影氛围？
9. 是否写明禁止项？
10. 是否存在过多无关细节导致模型混乱？
```

生图提示词必须稳定生成“关键帧”，不要写连续动作。

## 12.8 第八轮：图生视频提示词自检

检查问题：

```text
1. 是否只保留一个核心动作？
2. 是否只保留一个情绪变化？
3. 是否只保留一个镜头运动？
4. 是否要求保持角色、服装、场景一致？
5. 是否出现复杂动作？
6. 是否出现多人混乱互动？
7. 是否会导致角色换脸、换装、换场景？
8. 是否写了禁止项？
```

图生视频提示词必须避免：

```text
连续剧情
复杂动作
多次情绪跳变
大幅运镜
新增人物
新增道具
改变场景
```

# 13. 评分机制

每次拆解完成后，用 100 分制自评。

| 维度 | 分值 | 检查内容 |
|---|---:|---|
| 原文忠实度 | 20 | 是否不新增、不篡改、不遗漏 |
| 旁白拆分准确度 | 15 | 是否按标点、转折、情绪、信息点正确拆分 |
| 画面任务清晰度 | 15 | 每个画面是否有明确任务 |
| 情绪视觉化 | 15 | 是否把抽象情绪变成可见动作 |
| 站位与连续性 | 10 | 是否不越轴、不乱位、不跳场 |
| AI 可执行性 | 15 | 动作是否简单，镜头是否稳定 |
| 提示词完整度 | 10 | 生图 / 视频提示词字段是否完整 |

评级：

```text
90–100 分：可直接进入生产
80–89 分：可用，但需要小修
70–79 分：风险较高，需要重检
70 分以下：不建议进入生产，必须重拆
```

# 14. 常见错误与修正方式

## 14.1 错误：机械一句一镜

错误：

```text
每一句旁白只生成一个画面。
```

修正：

```text
按画面任务拆。
情绪句 2 个画面。
爆点句 3 个画面。
信息密集句拆成多个信息点。
```

## 14.2 错误：只写情绪，不写动作

错误：

```text
女主很伤心。
```

修正：

```text
女主低头忍泪，手指攥紧衣角，眼眶泛红，嘴唇轻微发抖。
```

## 14.3 错误：图生视频动作太复杂

错误：

```text
女主坐起，冲向男主，推开他，大喊，然后跑出病房。
```

修正：

```text
女主挣扎着撑住床边，缓慢抬头看向男主，眼神从震惊转为愤怒。
```

## 14.4 错误：转场生硬

错误：

```text
医院病房突然切到车祸现场。
```

修正：

```text
先拍女主嘴角血迹特写，血色反光匹配切到车祸现场红色警灯。
```

## 14.5 错误：人物站位混乱

错误：

```text
男主一会儿在女主左边，一会儿在女主右边。
```

修正：

```text
建立空间轴线：女主在画面左下方病床上，男主始终站在床尾右侧。
```

# 16. 可直接使用的母提示词

```text
请根据以下解说漫剧本，按照“画面任务”拆解成适合 AI 生图和图生视频的画面提示词。

拆解要求：

1. 不要按每句话机械拆镜，要按画面任务拆。
2. 先按逗号、句号、转折词、情绪爆点、信息点拆分旁白。
3. 每条旁白必须判断它承担的画面任务。
4. 画面任务包括：
   场景建立、人物状态建立、情绪放大、关系压迫、反应切镜、道具/证据细节、动作推进、爆点/反转/卡点、转场承接。
5. 普通信息句生成 1 个画面。
6. 情绪句生成 2 个画面：主画面 + 情绪放大画面。
7. 爆点句生成 3 个画面：铺垫 + 揭露 + 定格。
8. 每个画面必须包含：
   人物、场景、动作、表情、人物站位、景别、构图、机位、运镜、光影、情绪、道具/环境重点、转场方式。
9. 所有画面必须严格对应原文，不允许新增剧情、人物关系、关键道具或额外设定。
10. 情绪必须视觉化，不能只写“伤心、痛苦、绝望”，必须写成具体动作，例如：攥紧床单、眼眶泛红、嘴唇发抖、避开视线、肩膀颤抖。
11. 人物站位必须连续，不得越轴。
12. 场景和角色服装必须保持一致。
13. 每个图生视频提示词只能包含一个核心动作、一个情绪变化、一个镜头运动。
14. 动作要适合 AI 生成，优先使用轻动作和微表情，不要使用复杂打斗、多人混乱动作、大幅奔跑、夸张转身。
15. 每个大段落之间必须设计转场，优先使用视线、声音、动作、道具、光影、情绪进行承接。
16. 拆解完成后必须进行自检，并输出自检表。

输出表格字段：
编号｜对应旁白｜画面任务｜预计时长｜画面内容｜人物站位｜动作/表情｜景别/构图｜机位/运镜｜光影氛围｜转场方式｜生图提示词｜图生视频提示词｜自检备注。

自检要求：
1. 检查是否完整覆盖原文信息。
2. 检查是否新增原文没有的人物、道具、关系、剧情。
3. 检查每个画面是否有明确画面任务。
4. 检查情绪是否全部视觉化。
5. 检查人物站位是否连续，是否越轴。
6. 检查图生视频动作是否过于复杂。
7. 检查生图提示词是否包含角色、场景、动作、表情、构图、光影、禁止项。
8. 检查图生视频提示词是否只包含一个核心动作、一个情绪变化、一个镜头运动。
9. 按 100 分制给出自评分，并说明需要修正的问题。

整体风格统一为：
竖屏 9:16，AI 解说漫，2.5D-3D cinematic short drama style，北美短剧质感，电影级光影，情绪驱动，表演真实，画面连续。

以下是需要拆解的剧本：
```

# 18. 一句话总结

解说漫拆提示词，不是把文字逐句翻译成画面，而是把旁白拆成“观众必须看到的画面任务”。  
每个画面都要有任务，每个情绪都要能被看见，每个动作都要适合 AI 稳定生成，每个段落都要通过自检确认能进入生产。

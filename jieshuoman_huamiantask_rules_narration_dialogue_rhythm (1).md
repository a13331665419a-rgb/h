# 解说漫剧本画面任务拆解规则｜旁白对白节奏自检机制版

> 适用方向：AI 解说漫、AI 小说推文视频、AI 短剧旁白驱动画面拆解、可灵生图 / 图生视频提示词、Seedance / Vidu / 即梦类视频模型提示词前置拆解。

# 1. 核心原则

解说漫剧本不能按“每句话一镜”机械拆。

正确逻辑是：

```text
先读懂旁白内容
↓
按正常旁白语速估算读秒
↓
判断这段旁白 / 对白承担什么画面任务
↓
决定需要几个画面表现
↓
把抽象情绪转成可见动作
↓
保证人物站位、场景、道具、情绪连续
↓
再生成生图提示词和图生视频提示词
```

## 1.1 一镜必须有明确任务

每个画面都必须回答一个问题：

```text
这一镜是在建立场景？
还是在建立人物状态？
还是在放大情绪？
还是在制造关系压迫？
还是在揭露信息？
还是在给观众反应时间？
还是在承载人物对白？
还是在服务旁白停顿？
还是在服务转场？
还是在制造爆点卡点？
```

如果一镜无法回答这个问题，就说明它是无效镜头。

## 1.2 不要把旁白直接翻译成画面

错误逻辑：

```text
旁白：我很痛苦。
画面：女主很痛苦。
```

正确逻辑：

```text
旁白：我很痛苦。
画面：女主低头忍泪，手指死死攥住床单，嘴唇轻微发抖，肩膀压低，眼神避开男主。
```

## 1.3 画面要服务旁白节奏

解说漫画面不是独立短片，而是旁白的视觉承载。  
画面切换必须跟随：

```text
信息点
情绪点
转折点
悬念点
爽点
虐点
人物对白
语气停顿
付费卡点
```

## 1.4 先算声音时间，再拆画面数量

不能只看文字长度。  
同样 20 个字，如果是普通叙述，可能只需要 4–5 秒；如果是崩溃质问，可能需要 6–8 秒，因为需要停顿、反应和情绪呼吸。

核心规则：

```text
旁白 / 对白预计时长 = 正常语速读秒 + 情绪停顿 + 反应留白
画面数量 = 预计时长 ÷ 单镜合理时长
```

一般解说漫单镜建议：

```text
普通信息镜：2–3 秒
情绪特写镜：2.5–4 秒
对白口型镜：1.5–3 秒
听者反应镜：1.5–3 秒
爆点定格镜：3–5 秒
```

# 2. 标准工作流

拿到解说漫剧本后，按以下流程执行：

```text
第 1 步：通读全文，判断类型、核心情绪、叙事人称、声音来源
第 2 步：区分文本类型：解说旁白 / 人物对白 / 心声独白 / 环境音提示
第 3 步：按标点、转折词、情绪爆点、信息点拆分旁白
第 4 步：按正常语速估算每条旁白 / 对白的预计时长
第 5 步：判断每条文本承担的画面任务
第 6 步：根据读秒和画面任务，决定每条文本需要 1 个、2 个还是 3 个以上画面
第 7 步：生成画面信息表，包含画面任务、时长、站位、动作、情绪、转场
第 8 步：进行第一轮自检：原文忠实度、信息覆盖度、旁白读秒是否匹配
第 9 步：补齐缺失画面、删除无效画面、调整过短或过长画面
第 10 步：生成生图提示词
第 11 步：进行第二轮自检：角色、场景、站位、构图、光影、口型需求
第 12 步：生成图生视频提示词
第 13 步：进行第三轮自检：动作复杂度、镜头稳定性、旁白 / 对白节奏连续性
第 14 步：输出最终可用版本
```

# 3. 旁白拆分规则

## 3.1 按标点拆

优先按以下标点拆分基础旁白单元：

```text
逗号
句号
分号
问号
感叹号
冒号
破折号
省略号
```

示例：

```text
原文：我躺在急诊室里，刺鼻的消毒水味几乎让我窒息。
```

拆分为：

```text
1. 我躺在急诊室里
2. 刺鼻的消毒水味几乎让我窒息
```

## 3.2 按转折词拆

遇到以下词语，必须优先拆开：

```text
但是
可是
然而
没想到
直到
偏偏
真正
却
竟然
原来
此刻
下一秒
就在这时
```

示例：

```text
原文：我以为他会救我，可他却亲手把我推进深渊。
```

拆分为：

```text
1. 我以为他会救我
2. 可他却亲手把我推进深渊
```

第一段是期待，第二段是背叛反转，不能放在同一镜里。

## 3.3 按信息点拆

如果一句旁白中包含多个剧情信息，必须拆开。

示例：

```text
原文：昨晚的车祸明明是 Samantha 酒后无证驾驶造成的，可 Rhett 却向老师作证，说开车的人是我。
```

拆分为：

```text
1. 昨晚的车祸
2. Samantha 酒后无证驾驶
3. 撞上学校大门
4. Rhett 向老师作证
5. 他指认我是司机
```

每个信息点都必须有画面承载。

## 3.4 按情绪变化拆

如果一句旁白从一种情绪转到另一种情绪，必须拆开。

常见情绪变化：

```text
期待 → 失望
信任 → 背叛
委屈 → 愤怒
害怕 → 反抗
震惊 → 崩溃
羞辱 → 反杀
暧昧 → 危险
```

## 3.5 按声音来源拆

同一段文本里，旁白和人物对白不能混在同一个画面任务里处理。

必须先区分：

```text
解说旁白：由旁白声读出，不要求人物口型。
人物对白：角色在画面中说话，可能需要口型或说话动作。
心声独白：角色内心声音，可用角色静态情绪画面，不要求口型。
环境声音：门响、警报、心电仪、脚步声、玻璃破碎声，用于转场和节奏。
```

示例：

```text
旁白：我不敢相信他会这么说。
对白：“Scarlett，是你开的车。”
```

不能合成一个画面。应拆为：

```text
1. 女主听到前一半信息后的反应，承载旁白。
2. 男主说出台词的中近景，承载对白。
3. 女主瞳孔颤动的反应切镜，承接对白后的情绪。
```

# 4. 解说旁白与人物对白节奏规则

这一章是本版本新增的核心规则。  
所有画面拆解必须先进行声音读秒，否则容易出现以下问题：

```text
旁白 8 秒，画面只有 2 秒，观众还没听完画面就切走。
旁白 2 秒，画面给了 6 秒，节奏拖慢。
人物对白很长，却只给一个口型镜，AI 容易崩。
旁白和对白叠在一起，听感混乱。
爆点台词没有停顿，情绪起不来。
```

## 4.1 正常旁白语速估算

以下为生产估算值，用于拆镜和排时长，不是绝对标准。

| 文本类型 | 正常语速 | 情绪语速 | 适用情况 |
|---|---:|---:|---|
| 中文解说旁白 | 3.5–4.5 字/秒 | 2.8–3.5 字/秒 | 小说推文、解说漫、剧情旁白 |
| 中文人物对白 | 3.0–4.0 字/秒 | 2.4–3.2 字/秒 | 角色开口说话、质问、争吵、低语 |
| 英文解说旁白 | 2.2–2.8 words/秒 | 1.8–2.3 words/秒 | 北美短剧解说、英文 VO |
| 英文人物对白 | 1.8–2.5 words/秒 | 1.4–2.0 words/秒 | 角色对白、质问、哭腔、低声威胁 |

## 4.2 基础读秒公式

中文：

```text
旁白基础时长 = 中文字数 ÷ 4
情绪旁白时长 = 中文字数 ÷ 3.2
人物对白时长 = 中文字数 ÷ 3.2 + 口气停顿
```

英文：

```text
旁白基础时长 = 英文词数 ÷ 2.5
情绪旁白时长 = 英文词数 ÷ 2.0
人物对白时长 = 英文词数 ÷ 2.0 + 口气停顿
```

标点和情绪停顿需要额外加时：

| 停顿类型 | 建议加时 |
|---|---:|
| 逗号 / 短顿 | +0.2–0.4 秒 |
| 句号 / 信息结束 | +0.4–0.7 秒 |
| 问号 / 质问 | +0.5–0.9 秒 |
| 感叹号 / 情绪爆发 | +0.5–1.0 秒 |
| 省略号 / 犹豫 | +0.8–1.5 秒 |
| 爆点前停顿 | +0.8–1.5 秒 |
| 爆点后定格 | +1.5–3.0 秒 |
| 人物对白后的反应留白 | +0.5–1.5 秒 |

## 4.3 旁白时长反推画面数量

按预计读秒决定画面数量：

| 旁白 / 对白预计时长 | 推荐画面数量 | 处理方式 |
|---|---:|---|
| 1–2 秒 | 1 个短画面 | 情绪特写 / 道具细节 / 反应切镜 |
| 2–4 秒 | 1 个完整画面 | 普通信息、人物状态、单一动作 |
| 4–6 秒 | 2 个画面 | 主画面 + 情绪 / 反应 / 细节 |
| 6–9 秒 | 2–3 个画面 | 信息拆分 + 情绪递进 + 转场 |
| 9 秒以上 | 必须拆段 | 不建议一个旁白单元承载过长内容 |

规则：

```text
不是文字越多画面越多，而是读秒越长、信息越密、情绪越重，画面越需要拆。
```

## 4.4 旁白与画面时长匹配规则

每个画面时长不能只等于旁白读秒，还要留出视觉缓冲。

```text
画面时长 = 旁白读秒 + 0.3–0.8 秒视觉缓冲
```

特殊情况：

```text
爆点镜头：旁白结束后必须多留 1.5–3 秒定格。
转场镜头：可以比旁白短，但必须承接声音或动作。
情绪镜头：可以比旁白略长，让观众看清反应。
证据镜头：不能太短，至少 2 秒，让观众看清信息。
```

## 4.5 解说旁白的节奏类型

不同旁白类型要用不同语速和画面节奏。

| 旁白类型 | 语速 | 画面节奏 | 适合镜头 |
|---|---|---|---|
| 信息交代 | 正常偏快 | 画面清楚，不拖 | 场景建立、中景、道具镜 |
| 情绪倾诉 | 正常偏慢 | 多给特写和停顿 | 眼神、手部、呼吸、低头 |
| 悬念铺垫 | 慢一点 | 切镜少，压气氛 | 背影、门缝、局部特写 |
| 爆点揭露 | 先停再说 | 说完定格 | 特写、证据、反派表情 |
| 爽点反击 | 清晰有力 | 节奏上扬 | 抬眼、站直、反打特写 |
| 虐点崩溃 | 慢，句尾下沉 | 留白更长 | 泪眼、手抖、身体退缩 |

## 4.6 人物对白的节奏规则

人物对白不同于旁白。  
对白需要考虑：

```text
口型风险
说话动作
听者反应
情绪停顿
台词重音
对话回合
```

### 4.6.1 人物对白不要长时间只拍嘴

AI 生成视频中，长时间口型镜风险较高。建议：

| 对白长度 | 处理方式 |
|---|---|
| 1–2 秒短句 | 可以给正面或侧脸口型镜 |
| 2–4 秒台词 | 说话者中近景 + 轻微表情动作 |
| 4–6 秒台词 | 拆成说话者镜头 + 听者反应镜 |
| 6 秒以上台词 | 必须拆成多段，穿插反应、道具、压迫空间 |

### 4.6.2 对白镜头必须标注口型要求

每条人物对白需要明确：

```text
是否需要口型：需要 / 不需要 / 可用侧脸规避 / 可用反应镜覆盖
说话动作：嘴唇轻动、下颌绷紧、低声开口、短促质问
听者反应：僵住、低头、抬眼、手指收紧、呼吸停顿
```

示例：

```text
对白：“Why did you lie to the teacher?”
画面 1：Scarlett 侧脸近景，嘴唇轻动，声音沙哑，眼神盯着 Rhett，口型需要轻度匹配。
画面 2：Rhett 听到后没有立刻回答，眼神冷淡下压，避开她的视线，用反应停顿承接。
```

## 4.7 旁白与人物对白不能互相抢节奏

常见错误：

```text
旁白还没读完，人物对白已经开始。
人物对白说完没有反应，直接切下一句旁白。
旁白解释了对白已经表达过的信息，重复拖慢。
```

正确节奏：

```text
旁白铺垫 → 人物对白打断 / 推进 → 听者反应 → 旁白补充内心感受
```

或：

```text
人物对白 → 反应留白 → 旁白解释心理变化 → 下一镜推进剧情
```

## 4.8 旁白、对白、画面的三轨匹配

拆解时必须同时看三条轨道：

```text
声音轨：旁白 / 对白 / 环境声
画面轨：角色动作 / 表情 / 场景 / 道具
剪辑轨：切镜点 / 停顿点 / 转场点 / 卡点
```

每一镜都要写清：

```text
这一镜听到什么？
这一镜看到什么？
这一镜什么时候切走？
```

## 4.9 旁白压对白、对白压旁白的处理

一般不建议旁白和人物对白同时完整出现。特殊情况下可这样处理：

| 情况 | 处理方式 |
|---|---|
| 旁白介绍背景，角色无口型 | 旁白可覆盖静态情绪画面 |
| 角色正在说关键台词 | 暂停旁白，保留对白清晰度 |
| 对白很短，只作情绪刺点 | 可前后加旁白，但不要重叠 |
| 嘈杂环境中有人说话 | 可降低对白清晰度，作为环境声处理 |
| 内心独白覆盖人物沉默 | 可以，但画面中角色不能明显开口 |

## 4.10 爆点台词节奏

爆点台词不能快速掠过。必须三段处理：

```text
爆点前：0.8–1.5 秒停顿，画面压近或静止
爆点中：台词 / 旁白清晰说出核心信息
爆点后：1.5–3 秒定格，给观众消化和情绪记忆
```

示例：

```text
台词：“The driver was Scarlett.”

镜头 1：老师拿着证词停在病床前，病房安静半秒。
镜头 2：老师说出台词，镜头慢慢压近她手里的证词。
镜头 3：Scarlett 瞳孔微缩，嘴唇微张，画面定格 2 秒。
```

## 4.11 旁白时长与剪辑卡点规则

解说漫常见剪辑节奏：

```text
前 3 秒：必须建立冲突或异常处境。
每 5–8 秒：必须有一个新信息点或情绪变化。
每 12–18 秒：必须有一个小悬念、小反转或关系压迫升级。
每集结尾：必须有明确卡点画面，旁白 / 台词结束后保留定格。
```

短视频开头尤其不能慢：

```text
开头第 1 镜：直接给危机、羞辱、受伤、对峙、背叛、死亡、重生等强状态。
开头第 2 镜：补充人物状态或关系压迫。
开头第 3 镜：给问题或悬念，不要只做环境空镜。
```

# 5. 画面任务分类

所有画面必须归入以下任务类型之一。

| 画面任务 | 主要作用 | 常见使用场景 |
|---|---|---|
| 场景建立 | 让观众知道地点、时间、氛围 | 医院、学校、宴会、车祸现场、狼群大厅 |
| 人物状态建立 | 让观众知道角色当前身体和心理状态 | 受伤、虚弱、被羞辱、刚醒来、濒临崩溃 |
| 情绪放大 | 把抽象情绪变成可见动作 | 忍泪、发抖、攥紧衣角、眼神失焦 |
| 关系压迫 | 展示权力差、距离感、压迫感 | 男主俯视女主、众人围攻、家人冷眼 |
| 反应切镜 | 给观众看到角色接收信息后的变化 | 听到背叛、发现真相、看到证据 |
| 道具/证据细节 | 用物件承载剧情信息 | 手机、合同、病历、车钥匙、录音、血迹 |
| 动作推进 | 推动剧情向下一步发展 | 推门、走近、转身、递文件、抬头 |
| 对白承载 | 让人物台词清晰成立 | 质问、威胁、解释、命令、崩溃发问 |
| 听者反应 | 让对白后的心理变化成立 | 被误解、被背叛、被羞辱、听到真相 |
| 爆点/反转/卡点 | 强化爽点、虐点、悬念、付费点 | 身份曝光、背叛揭露、威胁、打脸 |
| 转场承接 | 保证镜头之间流畅过渡 | 视线、声音、光影、道具、动作匹配切 |

# 6. 画面数量判断规则

不是每条旁白都只生成一个画面。  
根据 **读秒 + 画面任务 + 情绪强度** 判断画面数量。

| 文本类型 | 预计时长 | 建议画面数量 | 拆解方式 |
|---|---:|---:|---|
| 普通环境信息 | 2–3 秒 | 1 个 | 场景建立 |
| 普通动作信息 | 2–3 秒 | 1 个 | 动作推进 |
| 普通人物状态 | 2–4 秒 | 1 个 | 人物状态建立 |
| 情绪句 | 4–6 秒 | 2 个 | 主画面 + 情绪放大 |
| 关系冲突句 | 4–7 秒 | 2 个 | 关系压迫 + 反应切镜 |
| 信息揭露句 | 5–9 秒 | 2–3 个 | 信息画面 + 证据细节 + 反应 |
| 人物对白短句 | 1–3 秒 | 1–2 个 | 说话者镜头 + 听者反应 |
| 人物对白长句 | 4–8 秒 | 2–3 个 | 说话者 + 反应 + 压迫关系 |
| 爆点反转句 | 5–10 秒 | 3 个 | 铺垫 + 揭露 + 定格 |
| 转场句 | 1–2 秒 | 1 个 | 道具 / 动作 / 视线 / 声音承接 |

## 6.1 普通信息句

只需要一个画面。

```text
旁白：她走进教室。
画面：女主推开教室门，教室里的学生纷纷回头看她。
```

## 6.2 情绪句

至少两个画面。

```text
旁白：我难过得快要喘不过气。
画面 1：女主站在人群边缘，身体微微后退。
画面 2：女主手指攥紧衣角，眼眶泛红，嘴唇发抖。
```

## 6.3 人物对白句

对白不只看台词内容，还要看“谁说、怎么说、谁听”。

```text
对白：“你为什么要骗老师？”
画面 1：女主撑着病床边缘，沙哑开口，嘴唇轻动。
画面 2：男主没有回答，冷漠垂眼，形成听者反应和关系压迫。
```

## 6.4 爆点句

必须三步处理。

```text
铺垫：女主听到某个信息后僵住。
揭露：男主 / 反派 / 证据完成关键信息揭露。
定格：女主或观众最需要记住的情绪特写。
```

# 7. 每类画面任务拆解标准

## 7.1 场景建立任务

必须包含：

```text
地点
时间
环境元素
主角位置
背景人物或背景运动
光影基调
整体氛围
```

错误：

```text
女主在医院。
```

正确：

```text
冷白急诊室里，女主半躺在病床上，头上缠着渗血绷带，输液架立在画面右侧，急诊帘半开，背景护士虚化经过，刺眼顶灯从上方压下来，整体氛围冷白、压抑、窒息。
```

## 7.2 人物状态建立任务

必须包含：

```text
身体状态
表情状态
手部动作
呼吸状态
姿态
服装 / 伤口 / 特殊标志
```

错误：

```text
女主很虚弱。
```

正确：

```text
女主脸色苍白，嘴唇干裂，肩膀细微发抖，右手死死攥住白色床单，头上的绷带边缘渗出血迹，眼神失焦却强撑着不哭。
```

## 7.3 情绪放大任务

禁止只写抽象情绪。

禁止：

```text
她很痛苦。
她很绝望。
她很伤心。
她很害怕。
她很委屈。
```

必须视觉化：

```text
她低头忍泪，睫毛轻颤，手指攥紧衣角，嘴唇轻轻发抖，眼神避开所有人的注视，肩膀向内收缩。
```

优先使用以下身体细节：

```text
眼神
睫毛
嘴角
嘴唇
手指
肩膀
呼吸
站姿
身体退缩
眼泪欲落未落
```

## 7.4 关系压迫任务

必须写清人物空间关系：

```text
谁高位
谁低位
谁靠近
谁后退
谁俯视
谁被围住
谁掌控空间
谁被空间压迫
```

示例：

```text
女主坐在病床左下方，身体低位虚弱；男主站在床尾右侧，身形挺直，冷漠俯视她，两人之间隔着病床尾栏，形成明显的情感距离和权力压迫。
```

## 7.5 对白承载任务

人物对白必须明确“说话动作”和“听者反应”。

必须包含：

```text
说话者是谁
说话者站位
台词情绪
是否需要口型
嘴唇 / 下颌 / 眼神动作
听者是谁
听者反应
台词后的停顿
```

错误：

```text
男主说：“是你开的车。”
```

正确：

```text
男主站在床尾右侧，身体没有靠近女主，冷淡垂眼说出“是你开的车”。他的嘴唇只轻微开合，语气平静到残忍。女主在病床左下方僵住，手指停止颤抖，眼神从震惊转为空白。
```

## 7.6 反应切镜任务

反应切镜不是重复剧情，而是呈现角色“接收信息后的变化”。

适合表现：

```text
听到背叛
听到真相
听到威胁
听到羞辱
看到证据
发现秘密
```

常用画面：

```text
女主瞳孔微缩
男主避开视线
反派嘴角轻轻上扬
母亲眼神动摇
围观者低声议论
主角手指突然停住
```

## 7.7 道具 / 证据细节任务

道具必须服务剧情，不允许随便添加。

可使用道具：

```text
手机屏幕
聊天记录
血迹
车钥匙
合同
录音笔
病历单
戒指
项链
破碎照片
门牌
校徽
酒杯
```

写法示例：

```text
手机屏幕特写，聊天记录停在“是她开的车”这一行，屏幕冷光映在女主苍白的脸上，她的手指悬在屏幕边缘，微微颤抖。
```

## 7.8 动作推进任务

动作必须简单、清楚、可生成。

适合 AI 的动作：

```text
抬头
低头
转身
走近
后退
推门
坐起
伸手
攥紧
松开
递出文件
看向某人
避开视线
```

不适合 AI 的动作：

```text
复杂打斗
多人混乱推搡
大幅奔跑追逐
复杂拥抱转身
摔倒翻滚
多人同步复杂动作
```

## 7.9 爆点 / 反转 / 卡点任务

爆点镜头必须三步走：

```text
1. 铺垫：观众意识到不对劲
2. 揭露：核心信息被说出或被看见
3. 定格：情绪特写或证据特写，形成停顿
```

示例：

```text
旁白：可真正陷害我的人，竟然是我最爱的人。

画面 1：女主听到真相后僵住，病房声音仿佛消失。
画面 2：男主冷漠站在床尾，避开她的眼神，没有任何愧疚。
画面 3：女主眼泪终于落下，画面定格在她不可置信的眼神上。
```

## 7.10 转场承接任务

每个大段落之间必须设计转场，不要硬切。

常用转场方式：

| 转场方式 | 用法 |
|---|---|
| 视线转场 | 女主看向门口 → 下一镜门被推开 |
| 声音转场 | 心电仪滴答声 → 车祸警报声 |
| 道具转场 | 血迹特写 → 车祸现场红色警灯 |
| 动作转场 | 手指攥紧床单 → 手握方向盘 |
| 光影转场 | 医院冷白灯 → 回忆中车灯强光 |
| 情绪转场 | 女主忍泪 → 回忆中崩溃大哭 |

# 8. 画面连续性与站位规则

解说漫虽然不是传统连续表演短剧，但人物站位必须稳定，否则 AI 视频会乱。

## 8.1 每个场景先建立空间轴线

进入新场景时，必须先定义：

```text
主角在画面哪一侧
对手在画面哪一侧
道具在哪一侧
门 / 窗 / 床 / 桌子的位置
镜头主要从哪一边拍
```

示例：

```text
医院急诊室空间轴线：
Scarlett 始终位于画面左下方病床上。
Rhett 始终站在床尾右侧。
输液架位于 Scarlett 右侧。
急诊门位于画面后方偏左。
镜头主要从 Scarlett 斜前方或床尾方向拍摄，不随意越轴。
```

## 8.2 禁止无理由换位

错误：

```text
镜头 1：男主站在女主右侧。
镜头 2：男主突然出现在女主左侧。
镜头 3：女主突然站到男主背后。
```

正确：

```text
除非剧情明确发生走位，否则同一场景内人物位置保持稳定。
```

## 8.3 反打镜头也要保持方向一致

如果镜头从女主特写切到男主特写，要保持视线方向。

```text
女主看向画面右上方。
男主看向画面左下方。
```

这样观众能理解两人正在对视。

# 9. AI 生成友好规则

## 9.1 优先微动作，不要复杂动作

AI 解说漫视频更适合：

```text
轻微抬眼
手指收紧
眼眶泛红
呼吸变急
嘴唇发抖
肩膀下沉
缓慢转头
轻微后退
```

不适合：

```text
快速打斗
复杂多人交互
高难度奔跑
多人同步动作
大幅度身体旋转
极端镜头翻转
```

## 9.2 每条视频提示词只允许一个核心动作

错误：

```text
女主坐起、哭泣、推开男主、冲出病房、回头大喊。
```

正确：

```text
女主挣扎着想坐起，右手撑住床边，眼神从震惊转为愤怒。
```

## 9.3 镜头运动要克制

推荐：

```text
缓慢推进
轻微拉近
固定镜头
轻微横移
轻微下压
慢慢虚化背景
```

不推荐：

```text
快速环绕
大幅摇镜
复杂航拍
突然变焦
旋转镜头
```

## 9.4 情绪靠表演和构图，不靠大动作

解说漫的强情绪主要靠：

```text
特写
眼神
手部细节
光影反差
人物距离
高低位压迫
背景虚化
声音节奏
```

# 10. 生图提示词生成规则

生图提示词的任务是生成稳定关键帧。

每条生图提示词必须包含：

```text
1. 画幅比例
2. 风格
3. 角色外貌
4. 场景
5. 人物站位
6. 动作
7. 表情
8. 景别
9. 构图
10. 机位
11. 光影
12. 情绪氛围
13. 声音节奏提示
14. 口型需求
15. 禁止项
```

标准结构：

```text
竖屏 9:16，AI 解说漫，2.5D-3D cinematic short drama style，北美短剧质感，电影级光影。

【声音节奏】
本镜承载旁白 / 对白 / 心声 / 环境声，预计时长 X 秒，语速正常 / 偏慢 / 情绪压低，是否需要口型。

【角色】
写清角色年龄、外貌、发型、服装、气质。

【场景】
写清地点、时间、环境、道具。

【画面内容】
写清人物正在做什么，谁站在哪里，谁看向谁。

【动作与表情】
写清手部、眼神、嘴角、肩膀、身体姿态。

【镜头】
写清景别、构图、机位、焦距倾向。

【光影】
写清冷暖、明暗、对比、饱和度、氛围。

【禁止项】
不要文字，不要水印，不要多余人物，不要变脸，不要变装，不要畸形手指，不要卡通感。
```

# 11. 图生视频提示词生成规则

图生视频提示词的任务不是重新讲剧情，而是让关键帧轻微动起来，并匹配旁白 / 对白节奏。

每条图生视频提示词只写：

```text
一个核心动作
一个情绪变化
一个镜头运动
一个环境动态
一条声音节奏说明
一组禁止项
```

标准结构：

```text
保持角色外貌、服装、场景、人物站位完全一致。

【声音节奏】
本镜承载旁白 / 对白，预计 X 秒。旁白语速正常 / 偏慢；如果是对白，角色嘴唇轻微开合，口型只需自然匹配，不夸张。

【角色动作】
角色进行轻微自然动作：写一个核心动作。

【情绪变化】
写眼神、呼吸、嘴角、肩膀、手指的细微变化。

【镜头运动】
缓慢推进 / 轻微拉近 / 横向微移 / 轻微下压 / 固定镜头。

【环境动态】
灯光轻微闪烁 / 窗帘轻动 / 输液管轻晃 / 背景人影虚化经过。

【禁止】
不要改变角色脸，不要改变服装，不要新增人物，不要大幅度动作，不要镜头乱转，不要卡通感，不要口型夸张。
```

## 11.1 旁白镜头的视频提示词

旁白镜头一般不需要人物开口。

```text
保持角色沉默，不要张嘴说话。通过眼神、手指、呼吸、肩膀细节承载旁白情绪。
```

适合：

```text
人物沉默特写
道具细节
环境压迫
回忆画面
听者反应
```

## 11.2 人物对白镜头的视频提示词

对白镜头要控制口型风险。

```text
角色嘴唇轻微自然开合，像在低声说话；下颌动作克制，眼神保持稳定，不要大幅摆头。
```

长对白不要全程正脸说话，应拆为：

```text
说话者中近景 → 听者反应 → 道具 / 空间压迫 → 说话者最后一句定格
```

# 12. 自检机制总览

本规则加入五轮自检机制。

```text
第一轮：文本拆解自检
检查是否漏拆、错拆、过度合并、过度拆碎。

第二轮：旁白 / 对白节奏自检
检查读秒、语速、停顿、对白口型、反应留白是否匹配。

第三轮：画面任务自检
检查每个画面是否有明确任务，是否服务原文。

第四轮：连续性自检
检查人物站位、场景、道具、转场是否连续。

第五轮：AI 可执行性自检
检查生图和图生视频提示词是否稳定、清楚、可生成。
```

自检不是最后才做，而是每完成一个阶段就检查一次。

# 13. 分阶段自检清单

## 13.1 第一轮：文本拆解自检

检查问题：

```text
1. 是否按标点拆分？
2. 是否按转折词拆分？
3. 是否按情绪变化拆分？
4. 是否按信息点拆分？
5. 是否区分了解说旁白、人物对白、心声独白、环境声？
6. 是否有重要信息被合并到一个过长画面里？
7. 是否把一句很短但情绪很重的旁白忽略了？
8. 是否出现旁白编号混乱？
9. 是否保留了原文顺序？
```

修正原则：

```text
信息多的句子要拆开。
情绪重的短句要放大。
转折句必须单独成镜。
人物对白必须单独处理。
不能改变原文顺序。
```

## 13.2 第二轮：旁白 / 对白节奏自检

检查问题：

```text
1. 是否计算了每条旁白 / 对白的预计读秒？
2. 是否标注了语速：正常 / 偏快 / 偏慢 / 情绪压低？
3. 旁白读秒是否和画面时长匹配？
4. 长旁白是否拆成多个画面？
5. 短旁白是否没有被过度拉长？
6. 情绪重的句子是否留出停顿？
7. 爆点前是否有 0.8–1.5 秒停顿？
8. 爆点后是否有 1.5–3 秒定格？
9. 人物对白是否标注了口型需求？
10. 长对白是否穿插听者反应，而不是一直拍说话者？
11. 旁白和对白是否互相抢节奏？
12. 人物对白后是否有反应留白？
```

修正原则：

```text
旁白 4–6 秒，至少拆 2 个画面。
旁白 6–9 秒，拆 2–3 个画面。
旁白 9 秒以上，必须重新拆段。
对白超过 4 秒，必须加入听者反应镜。
爆点句必须有停顿和定格。
旁白和对白一般不要完整重叠。
```

## 13.3 第三轮：画面任务自检

每个画面都必须回答：

```text
这一镜的画面任务是什么？
```

检查问题：

```text
1. 每个画面是否标注了画面任务？
2. 是否存在没有任务的空镜？
3. 情绪句是否有情绪放大画面？
4. 对白句是否有说话者和听者反应？
5. 爆点句是否有铺垫、揭露、定格？
6. 道具镜头是否真的服务剧情？
7. 反应切镜是否展示了角色变化？
8. 转场镜是否能自然接到下一镜？
```

如果某个画面不能归入任何任务类型，应删除或重写。

## 13.4 第四轮：原文忠实度自检

检查问题：

```text
1. 是否新增了原文没有的人物？
2. 是否新增了原文没有的关键道具？
3. 是否改变了人物关系？
4. 是否提前透露了后文信息？
5. 是否把旁白没说的动机强加给角色？
6. 是否把事实性信息改错？
7. 是否遗漏了旁白中的关键信息点？
8. 是否让对白表达了原文没有的意思？
```

原则：

```text
可以补充可视化细节。
不可以新增剧情事实。
可以强化情绪动作。
不可以改变角色动机。
可以设计转场。
不可以提前剧透。
```

## 13.5 第五轮：情绪视觉化自检

检查问题：

```text
1. 是否还存在“她很痛苦”“他很冷漠”这类抽象词？
2. 情绪是否变成了眼神、手指、呼吸、站姿、嘴角等可见动作？
3. 情绪变化是否有递进？
4. 画面情绪是否和旁白 / 对白语气一致？
5. 情绪是否过度夸张，变成不适合 AI 的大动作？
```

抽象词替换表：

| 抽象情绪 | 替换为可见动作 |
|---|---|
| 痛苦 | 手指攥紧、肩膀发抖、眉心压紧、呼吸变浅 |
| 绝望 | 眼神失焦、身体后退、手慢慢垂下 |
| 委屈 | 眼眶泛红、嘴唇发抖、低头忍泪 |
| 愤怒 | 下颌绷紧、眼神抬起、手指收紧 |
| 害怕 | 身体后缩、视线闪躲、呼吸急促 |
| 冷漠 | 眼神不动、嘴角无表情、身体不靠近 |
| 震惊 | 瞳孔微缩、动作停住、嘴唇微张 |

## 13.6 第六轮：站位与越轴自检

检查问题：

```text
1. 同一场景内人物位置是否稳定？
2. 角色是否无理由左右互换？
3. 反打镜头的视线方向是否正确？
4. 主角和对手的高低位关系是否明确？
5. 道具位置是否连续？
6. 场景入口、门、床、桌子等空间点是否混乱？
```

修正方式：

```text
为每个场景建立空间轴线。
每个镜头都写清谁在画面左侧，谁在画面右侧。
反打镜头保留视线方向。
除非剧情需要，不改变人物位置。
```

## 13.7 第七轮：节奏与时长自检

检查问题：

```text
1. 每个画面是否有预计时长？
2. 每个画面是否标注了对应声音来源？
3. 旁白长短是否和画面数量匹配？
4. 人物对白是否有口型 / 反应 / 停顿安排？
5. 情绪重的地方是否留出停顿？
6. 信息密集处是否切得太快？
7. 爆点前是否有铺垫？
8. 爆点后是否有定格？
```

参考时长：

| 画面类型 | 建议时长 |
|---|---:|
| 场景建立 | 2–3 秒 |
| 人物状态 | 2–3 秒 |
| 情绪特写 | 2–4 秒 |
| 关系压迫 | 3–4 秒 |
| 对白口型镜 | 1.5–3 秒 |
| 听者反应镜 | 1.5–3 秒 |
| 证据细节 | 2–3 秒 |
| 爆点定格 | 3–5 秒 |
| 转场镜 | 1–2 秒 |

## 13.8 第八轮：生图提示词自检

检查问题：

```text
1. 是否写明竖屏 9:16？
2. 是否写明风格？
3. 是否写明角色外貌和服装？
4. 是否写明场景和道具？
5. 是否写明人物站位？
6. 是否写明动作和表情？
7. 是否写明景别、构图、机位？
8. 是否写明光影氛围？
9. 是否写明本镜声音节奏与预计时长？
10. 如为对白镜，是否写明口型需求？
11. 是否写明禁止项？
12. 是否存在过多无关细节导致模型混乱？
```

生图提示词必须稳定生成“关键帧”，不要写连续动作。

## 13.9 第九轮：图生视频提示词自检

检查问题：

```text
1. 是否只保留一个核心动作？
2. 是否只保留一个情绪变化？
3. 是否只保留一个镜头运动？
4. 是否要求保持角色、服装、场景一致？
5. 是否标注旁白 / 对白预计时长？
6. 对白镜是否控制口型，不夸张张嘴？
7. 是否出现复杂动作？
8. 是否出现多人混乱互动？
9. 是否会导致角色换脸、换装、换场景？
10. 是否写了禁止项？
```

图生视频提示词必须避免：

```text
连续剧情
复杂动作
多次情绪跳变
大幅运镜
新增人物
新增道具
改变场景
口型夸张
旁白和对白混乱重叠
```

# 14. 评分机制

每次拆解完成后，用 100 分制自评。

| 维度 | 分值 | 检查内容 |
|---|---:|---|
| 原文忠实度 | 15 | 是否不新增、不篡改、不遗漏 |
| 旁白拆分准确度 | 10 | 是否按标点、转折、情绪、信息点正确拆分 |
| 旁白 / 对白节奏匹配 | 15 | 是否估算读秒、安排停顿、处理口型与反应 |
| 画面任务清晰度 | 15 | 每个画面是否有明确任务 |
| 情绪视觉化 | 15 | 是否把抽象情绪变成可见动作 |
| 站位与连续性 | 10 | 是否不越轴、不乱位、不跳场 |
| AI 可执行性 | 10 | 动作是否简单，镜头是否稳定 |
| 提示词完整度 | 10 | 生图 / 视频提示词字段是否完整 |

评级：

```text
90–100 分：可直接进入生产
80–89 分：可用，但需要小修
70–79 分：风险较高，需要重检
70 分以下：不建议进入生产，必须重拆
```

# 15. 常见错误与修正方式

## 15.1 错误：机械一句一镜

错误：

```text
每一句旁白只生成一个画面。
```

修正：

```text
按画面任务和读秒拆。
情绪句 2 个画面。
爆点句 3 个画面。
信息密集句拆成多个信息点。
旁白超过 6 秒必须考虑拆多个画面。
```

## 15.2 错误：旁白读秒和画面时长不匹配

错误：

```text
10 秒旁白只给一个 3 秒画面。
```

修正：

```text
先按正常语速估算读秒，再分配画面。
10 秒旁白至少拆成 3 个画面，分别承载信息、情绪、反应或转场。
```

## 15.3 错误：人物对白一直拍口型

错误：

```text
男主连续说 8 秒，镜头一直正脸拍嘴。
```

修正：

```text
拆成：男主说第一句 → 女主反应 → 证据细节 → 男主说最后一句。
```

## 15.4 错误：只写情绪，不写动作

错误：

```text
女主很伤心。
```

修正：

```text
女主低头忍泪，手指攥紧衣角，眼眶泛红，嘴唇轻微发抖。
```

## 15.5 错误：图生视频动作太复杂

错误：

```text
女主坐起，冲向男主，推开他，大喊，然后跑出病房。
```

修正：

```text
女主挣扎着撑住床边，缓慢抬头看向男主，眼神从震惊转为愤怒。
```

## 15.6 错误：转场生硬

错误：

```text
医院病房突然切到车祸现场。
```

修正：

```text
先拍女主嘴角血迹特写，血色反光匹配切到车祸现场红色警灯。
```

## 15.7 错误：人物站位混乱

错误：

```text
男主一会儿在女主左边，一会儿在女主右边。
```

修正：

```text
建立空间轴线：女主在画面左下方病床上，男主始终站在床尾右侧。
```

## 15.8 错误：旁白和对白互相抢声音

错误：

```text
旁白正在解释人物心理，同时人物也在说关键台词。
```

修正：

```text
先暂停旁白，让人物关键台词单独出现；台词后给听者反应，再用旁白补充心理变化。
```

# 17. 可直接使用的母提示词

```text
请根据以下解说漫剧本，按照“画面任务 + 正常旁白语速读秒 + 人物对白节奏”拆解成适合 AI 生图和图生视频的画面提示词。

拆解要求：

1. 不要按每句话机械拆镜，要按画面任务和读秒拆。
2. 先区分文本类型：解说旁白、人物对白、心声独白、环境声。
3. 再按逗号、句号、转折词、情绪爆点、信息点拆分旁白。
4. 每条文本必须估算预计声音时长：
   - 中文解说旁白按 3.5–4.5 字/秒估算。
   - 中文人物对白按 3.0–4.0 字/秒估算。
   - 英文解说旁白按 2.2–2.8 words/秒估算。
   - 英文人物对白按 1.8–2.5 words/秒估算。
   - 情绪句、质问句、崩溃句、爆点句要降低语速，并加入停顿。
5. 标点停顿规则：
   - 逗号加 0.2–0.4 秒。
   - 句号加 0.4–0.7 秒。
   - 问号加 0.5–0.9 秒。
   - 省略号加 0.8–1.5 秒。
   - 爆点前加 0.8–1.5 秒停顿。
   - 爆点后加 1.5–3 秒定格。
6. 每条旁白 / 对白必须判断它承担的画面任务。
7. 画面任务包括：
   场景建立、人物状态建立、情绪放大、关系压迫、反应切镜、道具/证据细节、动作推进、对白承载、听者反应、爆点/反转/卡点、转场承接。
8. 画面数量按预计声音时长判断：
   - 1–2 秒：1 个短画面。
   - 2–4 秒：1 个完整画面。
   - 4–6 秒：2 个画面。
   - 6–9 秒：2–3 个画面。
   - 9 秒以上：必须重新拆段。
9. 人物对白必须特别处理：
   - 短对白可用说话者镜头。
   - 超过 4 秒的对白必须穿插听者反应。
   - 必须标注是否需要口型：需要 / 不需要 / 侧脸规避 / 反应镜覆盖。
   - 人物对白后必须留出 0.5–1.5 秒反应空间。
10. 旁白和对白不能互相抢节奏。关键对白出现时，应暂停旁白；对白后用反应镜或旁白补充心理变化。
11. 每个画面必须包含：
   人物、场景、动作、表情、人物站位、景别、构图、机位、运镜、光影、情绪、道具/环境重点、声音节奏、口型要求、转场方式。
12. 所有画面必须严格对应原文，不允许新增剧情、人物关系、关键道具或额外设定。
13. 情绪必须视觉化，不能只写“伤心、痛苦、绝望”，必须写成具体动作，例如：攥紧床单、眼眶泛红、嘴唇发抖、避开视线、肩膀颤抖。
14. 人物站位必须连续，不得越轴。
15. 场景和角色服装必须保持一致。
16. 每个图生视频提示词只能包含一个核心动作、一个情绪变化、一个镜头运动。
17. 动作要适合 AI 生成，优先使用轻动作和微表情，不要使用复杂打斗、多人混乱动作、大幅奔跑、夸张转身。
18. 每个大段落之间必须设计转场，优先使用视线、声音、动作、道具、光影、情绪进行承接。
19. 拆解完成后必须进行自检，并输出自检表。

输出表格字段：
编号｜文本类型｜对应旁白/对白｜字数/词数｜语速类型｜预计声音时长｜画面任务｜画面预计时长｜画面内容｜人物站位｜动作/表情｜景别/构图｜机位/运镜｜光影氛围｜声音节奏｜口型要求｜转场方式｜生图提示词｜图生视频提示词｜自检备注。

自检要求：
1. 检查是否完整覆盖原文信息。
2. 检查是否新增原文没有的人物、道具、关系、剧情。
3. 检查是否正确区分旁白、对白、心声、环境声。
4. 检查每条文本是否估算声音时长。
5. 检查旁白 / 对白时长是否和画面数量匹配。
6. 检查人物对白是否有口型、听者反应、停顿处理。
7. 检查每个画面是否有明确画面任务。
8. 检查情绪是否全部视觉化。
9. 检查人物站位是否连续，是否越轴。
10. 检查图生视频动作是否过于复杂。
11. 检查生图提示词是否包含角色、场景、动作、表情、构图、光影、声音节奏、禁止项。
12. 检查图生视频提示词是否只包含一个核心动作、一个情绪变化、一个镜头运动。
13. 检查爆点前是否有停顿，爆点后是否有定格。
14. 按 100 分制给出自评分，并说明需要修正的问题。

整体风格统一为：
竖屏 9:16，AI 解说漫，2.5D-3D cinematic short drama style，北美短剧质感，电影级光影，情绪驱动，表演真实，画面连续。

```

# 19. 一句话总结

解说漫拆提示词，不是把文字逐句翻译成画面，而是把旁白和对白拆成“观众必须听清、看懂、记住的画面任务”。  
每个画面都要有任务，每段声音都要有读秒，每句对白都要有节奏和反应，每个情绪都要能被看见，每个动作都要适合 AI 稳定生成。
